The Property Editor

The property editor displays the properties of the currently selected object.  The following is true for the Property editor:  

  • Properties shown in gray display status information only.  As a general rule, properties shown in black can be edited.  Interrelated properties (see below) provide exceptions to this rule.
  • Properties exist for every object in the hierarchy including collection objects (i.e. Locations, All Fibers, NetworkViews).  Use collection object properties to view status information for, or make edits to, all lightpaths, demands, etc. in the collection (group).
  • Some properties are interrelated: 
    • Making changes to one property of an object may change the value of another property.  For example, the FixedLightPathRouting and FixedLightPathTimeSlots properties of demand objects are interrelated.  Setting the FixedLightPathTimeSlots property of a demand to True results in it's FixedLightPathRouting being set to True (if time slots are fixed then routing must be fixed also).  Likewise, setting the FixedLightPathRouting property of a demand to False results in the the FixedLightPathTimeSlots property being set to False (if lightpath routing is not fixed, then time slot routing cannot be fixed).
    • The value of one property of an object may determine if editing of other object properties are possible.  For example, if the Local property of a demand object is set to True, then none of the demand's path or protection related properties may be edited, as these properties do not apply to local demands.
