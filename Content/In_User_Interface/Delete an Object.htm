Delete an Object

To delete an object, click to select it and then press the "Delete" key.  When using the Object tree, you can also access the delete command by right-clicking on an object and selecting Delete from the pop-up menu. 

A message box will appear asking you to confirm your desire to delete.  After confirmation, the Modeler will delete the object and any objects related to it.  Related objects are objects that are connected to the deleted object in some way.  Examples of related objects include:

  • Any object that is a child of the deleted object.  When a network view is deleted, the locales within it are deleted also.
  • Any object that is linked to (routed over or through) the deleted object.  When a lightpath is deleted, any demands routed over it are also deleted.  When a location is deleted, any fibers or lightpaths connected to it are also deleted.
  • Any object that is a child of or is linked to a relative of the deleted object.  When a location is deleted, any lightpaths connected to it are also deleted and any demands routed over the related lightpaths will also be deleted.

As always, it is important to exercise caution when deleting information.  When you delete objects from your network file, you can replace them by reentry only.  Therefore, it is wise to frequently save a backup copy of your network file.
