Designing with Algorithms

This help section describes how to perform each step in the design process.  For help creating and opening network files, read the Network Files and Add Objects sections of this help document.

The Routing Algorithm - Modified Dijkstra

The Modeler uses a modified version of the Dijkstra algorithm to determine how to route Demand Paths over the LightPath layer. Each LightPath and Fiber has a mirrored set of directed Forward and Reverse Edges that connect its Start and End Locations together. Each Edge has its own Cost and Relaxed properties that are manipulated by various Modeler algorithms.

When a LightPath is relaxed both of its Edges are relaxed, along with all of the Fibers it is hosted by. When a Fiber is relaxed both of its Edges are also relaxed. When a Fiber Edge is relaxed any LightPath Edges hosted by it are also relaxed. When a LightPath Edge is relaxed prior to running the modified Dijkstra algorithm, the LightPath Edge will not be included in a Working or Protect Path between Start and End Locations.

Locations also have a relaxed property. When a Location is relaxed before running the modified Dijkstra Algorithm all LightPath Edges beginning or ending at the Location will not be included in a Working or Protect Path between Start and End Locations.

The Design Process

The process for creating a network design using the Modeler consists of:

  1. Use one of several methods to create a Network file.
  2. Route the network using the appropriate set of algorithms (mesh, ring, or a combination of both) as described below.
  3. Use various tools within the Modeler to analyze your design and make changes as needed.
  4. Use the Reporting Tool or the Network Data Tool to generate the desired Excel reports.

Mesh Routing

To route mesh demands, perform the steps as outlined below:

  1. Routing Preparation - complete entry of design information
  2. Run either the Route Demands algorithm or the Routing Optimization algorithm
  3. Reroute non-diverse demands using one of the following approaches:
  • Edit demand routing manually 
  • Adjust LightPath AdminCosts and re-route demands
  1. Optional: Run the Failure Analysis Algorithm

Ring Dimensioning

To route ring demands, perform the steps as outlined below:

  1. Routing Preparation - complete entry of design information 
  2. Run the Dimension Rings algorithm
  3. Optional: Run the Failure Analysis Algorithm
