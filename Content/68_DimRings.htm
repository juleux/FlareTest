Dimensioning Rings

The Ring Dimensioning algorithm routes demands with the goal of minimizing either the total network capacity (i.e. OC-48) or capacity-distance (i.e. OC-48-mi.). This minimization is achieved by repeatedly removing and re-routing each ring demand on the best path available to that demand. The algorithm takes into account both the working and protect capacity needed on each ring for that demand.  The algorithm will consider all defined rings, but depending on the topology and demand set, it will not necessarily use all rings.

You can run the Dimension Rings algorithm using either default or advanced dimensioning options.  Default ring dimensioning options are in effect from the moment you install the Modeler.  Their values are carefully selected to provide optimum routing results for a majority of applications.  Ring dimensioning options retain their default values unless you elect to edit them using the Advanced tab dialog.     

Note:  Advanced dimensioning options are reset to their default values upon closure of the Dimension Rings dialog.

Advanced dimensioning options are helpful in those rare situations where default parameters do not provide the desired results.  Each dimensioning option is described in the Advanced Dimensioning Options section of this help file.

Once you have determined which options to use, follow the steps below to initiate the dimensioning process:

  1. Create any additional objects or object properties necessary to define your network.  Refer to the list of Minimum Routing Requirements described in the Preparation for Routing section of this help file.
  2. Select Algorithms, Rings, Dimension Rings from the main menu. The following dialog will appear displaying an empty log:

Ring Dimensioning Dialog, Basic Tab

  1. If you want to execute the algorithm using default dimensioning options, proceed to the next step.  Otherwise, click on the Advanced tab and select the desired advanced dimensioning options.  When complete, click to return to the Basic tab.
  2. Click the Calculate button on the Basic tab to execute the dimensioning algorithm. 

Upon initiation, the Dimension Rings algorithm will perform the tasks listed below for all ring demands that are enabled for routing.  

  • Routes the demand working and protect paths over the lightpath layer.  Note that demands whose FixedLightPathRouting property is set to True or whose DoNotRoute property is set to True will not be routed.  Demands are routed based on lightpath admin costs.
  • Updates the lightpath TrafficWorking, TrafficProtectDedicated and TrafficProtectShared properties which are used to calculate the lightpath Capacity property.
  • Loads network statistics information into the log.

The appearance of network information in the log portion of the dialog is your signal that the dimensioning process is complete.  For tips on evaluating routing results, read the Evaluating Routing Results section of this help file.

Advanced Dimensioning Options

The Advanced tab of the Ring Dimensioning dialog is shown below.  By making selections in this dialog, you determine which factors take precedence during the dimensioning process.  Your selections will also influence the amount of time required for the algorithm to complete it's analysis.  Use the description of each option (included below) to determine the desired settings.  When your selections are complete, return to the Basic tab and click on the Calculate button to initiate the dimensioning process.

Ring Dimensioning Dialog, Advanced Tab

A description of each advanced routing option is provided below.  Any changes you make to option values are retained until the Ring Dimensioning dialog is closed. 

Minimize LightPath

By default, Modeler makes routing decisions with the goal of minimizing total lightpath capacity.  As an alternative, Modeler can make routing decisions with the goal of minimizing total lightpath capacity distance. 

Clear Routing and Statistics

This option determines whether existing routing information will be deleted prior to running this algorithm.  As the algorithm runs, each potential solution is compared to the 'Initial Solution' and the least cost result is retained.  This result becomes the 'Initial Solution' to which subsequent solutions are compared.  

When this option is unchecked (default), the solution that existed prior to running the algorithm is used as the 'Initial Solution'.  When this option is checked, the algorithm does not use the pre-existing solution as it's 'Initial Solution'.  Use this option when you want to route all of the demands to arrive at a new solution.  For example, in situations, such as adding more demand to the network, that will naturally require a more expensive solution than the pre-existing one.

Delete Rings Without Traffic

When checked, rings without traffic will be deleted after the dimensioning process is complete.

Iterations

During each iteration, solution's capacity/capacity distance is calculated and compared to that of the benchmark (lowest capacity/capacity distance) solution.  The solution with the lowest capacity/capacity distance becomes the new benchmark to which future solutions will be compared.  Demands are randomly sorted between iterations.  Hence, the more iterations that the algorithm is run, the more demand sequences will be tried and the greater the chance of finding the lowest capacity/capacity distance solution.  To run the algorithm  indefinitely, set the number of iterations to zero.  To stop running the algorithm, click the Stop button on the progress dialog.
