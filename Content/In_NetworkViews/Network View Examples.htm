Network View Examples

Some of the basic functionality of network views is presented in the three examples provided below.  Each example uses the default property settings of Modeler's Fiber view, except where noted.  

Fiber Layer

The first example below shows the fiber layer of the map.  Fibers appear as dark red lines connecting locales .  Multiple fibers can exist between any two locales. 

  • When you click on a fiber to select it:

    • The fiber changes to a bold red line. 

    • Selection handles appear.  Click and drag a handle to pull a fiber aside, forming an arc (endpoints remain in place).  Any additional fibers underneath will be revealed.

  • When you select a lightpath (using the object tree), the fibers over which the lightpath travels appear as bold red lines. 

  • When you select a ring (using the object tree), the fibers over which the ring's lightpaths travel appear as bold red lines.  

  • When you select a routed demand (using the object tree), the fibers over which the demand is routed appear as bold red lines, solid for the work path, dashed for the protect path (if present). 

  • When you select a conduit (using the object tree), the fibers associated with the conduit (in other words, fibers whose ConduitId property contains the id of the selected conduit) appear as bold red lines.

LightPath Layer

The second example below shows the lightpath layer of the map superimposed on the fiber layer.  LightPaths appear as dark green lines connecting locales .  Multiple lightpaths can exist between any two locales. 

  • When you click on a lightpath to select it:

    • The lightpath changes to a bold green line.

    • Selection handles appear.  Click and drag a handle to pull a lightpath aside, forming an arc (endpoints remain in place).  Any additional lightpaths or fibers underneath will be revealed.

  • When you select a routed demand (using the object tree), the lightpaths over which the demand is routed appear as bold green lines. 

  • When you select a ring (using the object tree), the lightpaths which form the ring appear as bold green lines. 

  • When you select a conduit (using the object tree), the lightpaths associated with the conduit (in other words, lightpaths whose ConduitId property contains the id of the selected conduit) appear as bold green lines. 

  • When you select a VPN (using the object tree), the lightpaths assigned to the VPN  (in other words, lightpaths whose VPN property contains the id of the selected VPN) appear as bold green lines.   

 

Note:  In exception to default Fiber view settings, lightpaths are shown in this example.  To display lightpaths, set the LightPathRouting property of the view to Direct.

Demand Layer

The third example below shows the demand layer of the map superimposed on the fiber and lightpath layers.  Demands appear as blue lines.  Multiple demands can exist between any two locales.  When selected, demands function in the same way as fibers and  lightpaths - they appear as bold lines with selection handles to enable dragging.

As this example illustrates, the more information a view contains, the more difficult it becomes to identify and select individual objects. The following Modeler tools are particularly useful when working with object-dense views:

  • Spread Links tool - object layers can be spread (formed into an arc) by clicking on the Spread links icon.  To re-layer (straighten) links that have been spread, click on the Straighten links icon.  
  • Pan and Zoom window - to navigate within large views and change their scale.
  • Expand View Proportionately tool - this tool changes the relative position of a view's locales by a user specified factor.  To expand (or contract) a view, click on the icon in the Main Toolbar and enter a value for the expansion factor. 

 

Note:  In exception to default Fiber view settings, demands and  lightpaths are shown in this example.  To display demands and lightpaths, set the LightPathRouting property and the DemandRouting property of the view to Direct.  To display demand quantity, set the DemandLabel property of the view to Quantity.
