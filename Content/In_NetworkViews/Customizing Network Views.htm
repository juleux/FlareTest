Customizing Network Views

Once a network view has been created, you can customize it by editing its properties to suit your needs.  Network views can be customized in the following ways:

Description Instructions
Display various types of fiber, lightpath, demand, and location data.
  • To select data for display, select a network view and set its FiberLabel (LightPathLabel, DemandLabel, or LocaleLabel) property using the pull-down menu.
Label Type Selection Description
 Fiber None Label hidden
Name Fiber Name
Traffic Demand traffic over fiber in STS-1 units in the following format:  Working:ProtectDedicated:ProtectShared:Available
Capacity Sum of the working and protect traffic for the lightpaths traveling over this fiber
AdminCost Fiber AdminCost
Length Fiber Length
ConduitIDs The list of conduit id numbers associated with this fiber.  Multiple ids are separated by colons.
ConduitIDDescriptions The description of each of the conduit ids associated with this fiber.  Multiple descriptions are separated by colons.
 LightPath None Label hidden
Name LightPath Name
Length LightPath Length
AdminCost LightPath AdminCost
Traffic Demand traffic over lightpath in STS-1 units in the following format:  Working:ProtectDedicated:ProtectShared:Available
Capacity Sum of the working and protect traffic in lightpath Type units
CapacityMaximumType Sum of the working and protect traffic in MaximumType units (the largest type present in the lightpath collection)
CapacityIncremental The difference between the actual capacity and the minimum capacity of a lightpath.  When this label is selected any lightpath with an incremental capacity greater than zero will be highlighted in the network view.
CapacityMinMaxCapIncUtil Multiple capacity related properties of a lightpath displayed in the format - MinimumCapacity:MaximumCapacity:Capacity:IncrementalCapacity:Utilization
CapacityOverMaximum The difference between the maximum capacity and the actual capacity of a lightpath
Domain The ring, if any, that this lightpath is associated with.
RingStackCount The stack count of the ring associated with this lightpath
Utilization The percentage of this lightpath's capacity in use.  When this label is selected, each lightpath in the network view is displayed using a color that corresponds to it's utilization value:
  • Green - Utilization = 0
  • Blue - Less than or equal to 25
  • Yellow - Less than or equal to 50
  • Orange - Less than or equal to 75
  • Dark Red - Less than or equal to 100
  • Red - Greater than 100
ConduitIDs The list of conduit id numbers associated with this lightpath.  Multiple ids are separated by colons
VPN The id number of the Virtual Private Network this lightpath is associated with.
FailureAnalysisAvailable This label is useful when you want to look at a network view during Manual Failure Analysis to evaluate the impact of one or more failures on the network.  Select this label to view the quantity of time slots of each signal type that are available after a manual failure or repair.  Data is in the format of Type1:Type3:Type12:Type48:Type192 where
  • Type1 = OC1_STS1_VC3_DS3_E3_AU3
  • Type3 = OC3_STS3_STM1_VC4
  • Type12 = OC12_STS12_STM4
  • Type48 = OC48_STS48_STM16
  • Type192 = OC192_STS192_STM64

Name Demand Name

 Locale None Label hidden
Name The Name of the location associated with this locale

Select one of several methods for displaying fiber, lightpath and demand routing information.
  • To select a method of displaying fiber, lightpath and demand routing information, select a network view and set its FiberRouting (LightPathRouting, DemandRouting) property using the pull-down menu. 
Routing Type Selection Description
Fiber None Fibers are not shown.
Direct Each fiber is shown as a line from its start location to its end location.
 LightPath None LightPaths are not shown.
Direct Each lightpath is shown as a line from its start location to its end location.
OverFibers The fibers over which a lightpath travels are highlighted when the lightpath is selected.  The view's FiberRouting property must be set to Direct to take advantage of this feature. 
 Demand None Demands are not shown.
Direct Each demand is shown as a line from its start location to its end location.
OverFibersAndLightPaths The fibers and lightpaths over which a demand is routed are highlighted when the demand is selected.  The view's FiberRouting and/or LightPathRouting property must be set to Direct to take advantage of this feature. 
Display fibers, lightpaths, and demands as straight or curved lines.
  • To display the fibers within a view as curved lines, select the view and set its FibersAsBeziers (LightPathsAsBeziers, DemandsAsBeziers) property to True.  Handles will appear on each fiber.  To form a curve, drag one or both handles on a fiber to the desired coordinates.  Fiber endpoints remain fixed.
  • To curve (spread) overlapping fibers, lightpaths, and/or demands within a view, click on the Spread Link Items icon .  The FibersAsBeziers (LightPathsAsBeziers, DemandsAsBeziers) property of a layer must be set to True for spreading to occur.  Use the view's SpreadFactor property to control the severity of the resulting curves (the relative distance of each curve's peak from its endpoints).  
  • To straighten all fibers, lightpaths and/or demands within a view, click on the Straighten Link Items icon .
Display or hide individual locales.
  • When a view is created, its locales are visible by default.  To hide a locale, click on the desired locale in the object tree and set its Visible property to False.
Reposition locales within a view.
  • To move a single locale, click on the locale in the network view window and drag it to the desired coordinates.
  • OR: Click on the desired locale in the object tree and edit its X and Y (coordinate) properties.
  • To move a group of two or more locales, first select the desired locales by holding down the Shift key and clicking on each locale's icon within the view.  Next, click on one of the selected locales and drag to the desired coordinates.
  • To align a group of two or more locales, first select the desired locales by holding down the Shift key and clicking on each locale's icon within the view. Next, click on an Alignment tool icon (left: , right: , top: , or bottom: ) in the Main Toolbar.
  • To change the distance between every locale within a view, click on the Expand View Proportionately icon  in the Main Toolbar.  Enter the factor by which you wish to expand or contract the view.  The background image will be expanded/contracted as well.
  • All of the locales within a view can be repositioned as a group using the Auto Layout Locales tool .  This tool uses an algorithm to create a best-case view automatically.
Change the scale of a view.
  • To increase a view's scale, click on the view in the object tree and set its Scale property to a number greater than one.  Enter a number between 0.1 and 1 to decrease a view's scale.
  • OR: Change a view's scale by selecting the view and clicking on a Zoom icon (increase scale: , decrease scale: , original scale: , or scale to fit in window: ) in the Main Toolbar.
  • OR: Change a view's scale using the Pan and Zoom Window tool located in the upper left corner of the view window.  
Edit the colors and icons used to represent objects.
  • To change the color used to highlight selected objects within a view, set its SelectionColor property using the pull-down menu.  To add a shadow to selected objects within a view, set its SelectionShadow property to True.
  • Each locale you create is assigned an image, referred to as an icon, which is used to represent it within the network view.  When a locale is created, it's IconFilename property is set based on the Type property of the location it is tied to.  To change a locale icon, click on the desired locale in the object tree and set its IconFileName property by entering a filename with the .ico extension (or click on ... to browse for a file).
  • To change fiber (lightpath, demand) color, select a network view and set its FiberColor (LightPathColor, DemandColor) and FiberColorSelected (LightPathColorSelected, DemandColorSelected) properties using the pull-down menus.
  • When the Failure Analysis window is open, you can select an object (i.e. demand, lightpath) listed in the Failure Analysis Results tab and the fibers and lightpaths associated with it will be highlighted in the network view.  To change the highlight color of failed fibers (lightpaths), select a network view and set its FiberColorFailed (LightPathColorFailed) property using the pull-down menu.  To change the highlight color of fibers (lightpaths) used to restore a demand, select a network view and set its FiberColorOnRestoration (LightPathColorOnRestoration) property using the pull-down menu.
Change the appearance of a view's background.
  • To change a view's background color, click on the view in the object tree and set its BackgroundColor property using the pull-down menu.
  • To change a view's background image, click on the view in the object tree and set its BackgroundFileName property by entering a filename (or click on ... to browse for a file).  To scale a view's background image, edit its MapSizeWidth property.
  • To add a grid to a view, click on the view in the object tree and edit its GridStyle property using the pull-down menu. To change grid color, edit the view's GridColor property.
