Create an Excel File for Import

Network information can be entered into Excel using the format described below and then imported into the Modeler.  Network data exported to Excel is stored using this same format.  

A network data file stores data in the following types of worksheets:

  • Locations
  • Fibers
  • Rings
  • LightPaths
  • Demands

To create a network data file, create an Excel file with the desired worksheets and enter data in the format described below.  Guidelines for creating worksheets include:

  • Multiple worksheets of the same type can be present in a single network data file.  Each worksheet of a given type represents a group and has a unique name.  Groups can be used in any number of ways - they are often used to represent phases of network implementation, geographic areas, or various design approaches to be analyzed. 

  • In order for Modeler to properly read in your data, each worksheet must be given a name beginning with its type (Locations, Fibers, etc.) followed by an optional group name.  Examples include: Locations, Fibers Phase2, Demands (3), LightPaths NorthEast.  Group names should be limited to alphanumeric characters.

  • A network data file need not contain every type of worksheet.  Worksheets can be combined in a multitude of ways.  Base your selections on the type of network (mesh or ring) being designed and on the ways in which you want to group information.  For a list of the information required by Modeler's routing algorithms, read the Preparation for Routing section of this help document.

  • The data Modeler looks for in each type of worksheet is described in the tables below.  Data is entered in columns with predefined header names.  Some columns are required and must be present for the file to open properly. 

  • Additional information may be included in a worksheet as desired.  The Modeler treats additional columns as described in the UserProperties section below.

User Properties

The Modeler's UserProperties feature enables you to include additional data in a network data file.  The additional data is not directly used by the Modeler.  It is pass-through information, visible during the design process, present in Modeler generated reports, and exportable to other applications.  

To create a UserProperty:

  1. Select the desired Locations, Fibers, Rings, LightPaths, or Demands worksheet(s).
  2. Add the desired column(s) of information with a header entry in the first row.

Tips for using UserProperties include:

  • They can't be created to the right of the Fiber1...FiberN columns on the LightPaths sheets.
  • They only carry text values in and out of the Modeler's object model, not Excel formulas.
  • They appear in reports that present data from the worksheet they reside in (i.e. LightPath UserProperties will be included in the LightPath Traffic report).
  • To keep an Excel worksheet column from being read into UserProperties, either leave the first header row blank or simply use the character '#' as the first character of the header name.
  • They can be created and editing using the Property editor.

Locations Worksheet

Locations worksheet names must begin with the word Locations.  Any additional text following Locations is considered a group name.  Locations worksheets contain all or part of the information described in the table below.  Each worksheet of this type has required data.  The names of columns containing required data are highlighted.  Columns do not have to be arranged in any specific order, but must be properly named.

Column Name(s) Description
Name Enter a unique name for each location.
LongName A location descriptor (non-unique) used for documentation. 
Locked Enter True to protect a location from being deleted.  If this column is absent from the worksheet, this property defaults to a value of False.
Longitude, Latitude Optional input. Provides relative map placement within the Modeler.
CLLI, Building, Hub Optional input. Location information used for documentation.
Type Optional input. Determines the icon used to represent the location in each network view.  Valid entries include:  SN16000, SN9000, SN3000, Switch.
PermitThroughTraffic Optional input. Enter the type of lightpath that can be routed through a location.  Valid entries include:
  • Any LightPath
  • Express LightPaths
  • None

Fibers Worksheet

Fibers worksheet names must begin with the word Fibers.  Any additional text following Fibers is considered a group name.  Fibers worksheets contain all or part of the information described in the table below.  Each worksheet of this type has required data.  The names of columns containing required data are highlighted.  Columns do not have to be arranged in any specific order, but must be properly named.

Column Name(s) Description
Name Enter a unique name for each fiber.  
LongName A fiber descriptor (non-unique) used for documentation. 
Locked Enter True to protect a fiber from being deleted.  If this column is absent from the worksheet, this property defaults to a value of False.
StartLocation, EndLocation Enter the name of the location where each fiber starts/ends.  Values must match the Name property of a location. 
Cost Use to assign cost to a fiber, with units of your choice (fiber length is typically used).  Used by the Modeler for routing purposes.  If this column is absent from the worksheet, this property defaults to a value of 1.
Length Enter a value for fiber length. If this column is absent from the worksheet, this property defaults to a value of 1.
ConduitIDs Enter identifiers, separated by colons, for the conduits the fiber passes through. If this column is absent from the worksheet, this property is set to the value of the fiber's Name property.

Rings Worksheet

Rings worksheet names must begin with the word Rings.  Any additional text following Rings is considered a group name.  Rings worksheets contain all or part of the information described in the table below.  Each worksheet of this type has required data.  The names of columns containing required data are highlighted.  Columns do not have to be arranged in any specific order, but must be properly named.

Column Name(s) Description
Name Enter a unique name for each ring.  
LongName A ring descriptor (non-unique) used for documentation. 
Locked Enter True to protect a ring from being deleted.  If this column is absent from the worksheet, this property defaults to a value of False.
Architecture Enter the type of ring architecture.  Valid entries include:
  • BLSR2F_MSSSPRING2F
  • BLSR4F_MSSSPRING4F (default)
  • ASYM4F
  • UPSR_SNCP
Type Enter the type of lightpath that can be assigned to each ring.  Valid entries include:
  • OC3_STM1
  • OC12_STM4
  • OC48_STM16
  • OC192_STM64 (default)

LightPaths Worksheet

LightPaths worksheet names must begin with the word LightPaths.  Any additional text following LightPaths is considered a group name.  LightPaths worksheets contain all or part of the information described in the table below.  Each worksheet of this type has required data.  The names of columns containing required data are highlighted.  Columns do not have to be arranged in any specific order - with one exception, the Fiber1..FiberN columns must be the rightmost columns in the worksheet.  Columns must be properly named.

Column Name(s) Description
Name Enter a unique name for each lightpath.  
LongName A lightpath descriptor (non-unique) used for documentation. 
Locked Enter true to protect a lightpath from being deleted.  If this column is absent from the worksheet, this property defaults to a value of False.
StartLocation, EndLocation Enter the name of the location where each lightpath starts/ends. Values must be defined as fiber termination points, in other words they must match the StartLocation/EndLocation property of a fiber.
Cost Enter a value to assign cost to a lightpath, with units of your choice (fiber length is typically used).  Used by the Modeler for routing purposes. If this column is absent from the worksheet, this property defaults to a value of 1.
Length Enter a value for lightpath length. If this column is absent from the worksheet, this property defaults to the sum of the Length of each fiber in the lightpath.
ConduitIDs Enter identifiers, separated by colons, for the conduits the lightpath passes through. If this column is absent from the worksheet, this property defaults to a list of the unique ConduitIds for each fiber in the lightpath.
VPN Enter the numeric Id of the Virtual Private Network (if any) associated with this lightpath.  If this column is absent from the worksheet, this property defaults to a value of 0.
Domain Enter the name of the ring (if any) associated with this lightpath.  
Type Enter lightpath type.  Valid entries include:
  • OC-3
  • OC-12
  • OC-48
  • OC-192 (default)
  • STM-1
  • STM-4
  • STM-16
  • STM-64

The Modeler works with standard groupings rather than individual lightpath types.  When a network file is opened, the Modeler converts each Type entry to the appropriate group.  Examples of groups include:

  • OC3_STM1
  • OC12_STM4
  • OC48_STM16
  • OC192_STM64
CapacityMinimum The minimum capacity of a lightpath.  Enter the amount of any pre-existing lightpath capacity.  Defaults to a value of 0 which disables capacity checks.
CapacityMaximum The maximum capacity of a lightpath. This value is used by routing algorithms to limit the capacity of a lightpath.  Defaults to a value of 0 which disables capacity checks.
TrafficProtectShared The amount of shared path protect traffic routed over this lightpath in STS-1 (SONET) or STM-1 (SDH) units depending on network type.
ProtectSharedTV The quantity of time slots of each signal type assigned to the demand's protect shared path in the format of Type1:Type3:Type12:Type48:Type192 where
  • Type1 = OC1_STS1_VC3_DS3_E3_AU3
  • Type3 = OC3_STS3_STM1_VC4
  • Type12 = OC12_STS12_STM4
  • Type48 = OC48_STS48_STM16
  • Type192 = OC192_STS192_STM64
Fiber1, ... Fiber11 Identifies the fiber(s) that the lightpath travels over.  Enter a fiber name in the Fiber1 column.  Enter additional fiber names in Fiber2  through Fiber11 as required.  Values must match the Name property of a fiber and provide a valid path from the lightpath's StartLocation to its EndLocation.

Demands Worksheet

Demands worksheet names must begin with the word Demands.  Any additional text following Demands is considered a group name.  Demands worksheets contain all or part of the information described in the table below.  Each worksheet of this type has required data.  The names of columns containing required data are highlighted.  Columns do not have to be arranged in any specific order, but must be properly named.

Column Name(s) Description
Name Enter a unique name for each demand.  
LongName A demand descriptor (non-unique) used for documentation. 
Locked Enter true to protect a demand from being deleted.  If this column is absent from the worksheet, this property defaults to a value of False.
StartLocation, EndLocation Enter the name of the location where each demand starts/ends.  Values must be defined as LightPath termination points - they must match the StartLocation/ EndLocation property of a LightPath.
Scope Enter the scope of each demand.  Modeler's routing algorithms factor scope into the demand routing process.  Valid entries include:
  • Mesh 
  • Ring 
VPN Enter the numeric Id of the Virtual Private Network (if any) associated with this demand.  If this column is absent from the worksheet, this property defaults to a value of 0.
Type Enter the demand type. Valid entries include: 
  • AU3, DS-3, VC3, VC4, E3
  • STM-1, STM-4, STM-16, STM-64
  • STS-1, STS-3, STS-12, STS-48, STS-192
  • OC-1, OC-3, OC-12, OC-48, OC-192

The Modeler works with standard groupings rather than individual demand types.  When a network file is opened, the Modeler converts each Type entry to the appropriate group.  Examples of groups include:

  • OC1_STS1_VC3_DS3_E3_AU3 (default)
  • OC3_STS3_STM1_VC4
  • OC12_STS12_STM4
  • OC48_STS48_STM16
  • OC192_STS192_STM64
Quantity Enter demand size (must be greater than or equal to 0) in units specified by the Type field. If this column is absent from the worksheet, this property defaults to a value of 1.
ProtectionType Enter the protection type.  Valid entries include: 
  • Unprotected (working only) 
  • OnePlusOne (working and protect) 
  • SharedPath (working and protect)
DiverseGroup Enter the name of the Diverse Group this demand is a member of.   Each demand within a diverse group is routed diversely from the every other demand in the group.  If this column is absent from the worksheet, this column defaults to a null value and no diverse groups will be present.
NodeProtection Used to factor in node protection.  Enter True when node protection is present.  Enter False when it is absent. If this column is absent from the worksheet, this property defaults to a value of False.
Description A demand descriptor used for documentation. 
Fixed TimeSlots When True, prevents routing algorithms from making changes to existing time slot assignments.
Working, Protect Used to specify lightpaths for use as working and protect paths.  Enter lightpath names in order, separated by commas. Values must match the Name property of a lightpath and provide a valid path from the demand's StartLocation to its EndLocation.
Working Starting TimeSlots, Protect Starting TimeSlots Used to specify time slots for use as working and protect paths for mesh demands.  Starting time slot information is displayed in the following format:  Starting Time Slots for Hop1 (one for each demand quantity, separated by colons), Starting Time Slots for Hop2, ... An example is shown below.

If you would like to assign consecutive blocks of time slots for demands with a quantity greater than one, use the sample format below:

 
Fixed Routing When True, prevents routing algorithms from making changes to working and protect paths.
Routing Notes Contains text messages from routing algorithms.
Broadleaf Source Reroute Used to specify the state (on/off) of Broadleaf Source Rerouting.
Broadleaf Reversion Enter the reversion type.  Valid entries include: 
  • Disabled 
  • Manual 
  • Auto
Broadleaf Trunk Restoration Used to specify the state (on/off) of Broadleaf Trunk Restoration.
