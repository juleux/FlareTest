Import Network Data from

is a tool to help operators collect and manage live network data.  When you establish a connection to a server, you can import data from an existing network into the Modeler and use it to create a network file.

To import data from , select File, Import, from from Modeler's main menu.  The Modeler will create a new network file to contain data imported from .  If a file is currently open, the Modeler will prompt you to save or exit it before displaying the Import Network from dialog.  The dialog is composed of multiple tabs.  Each tab represents a step in the import process.  The process begins with the Select Data Source tab and progresses from left to right until complete.

Note:  The Import tool  requires an advanced license from and the proper installation of the Oracle Data Provider.  For the files and instructions necessary to enable you to import data from r, contact your local representative.

Select a Data Source and Get Data

There are two methods of obtaining data from :

  • Direct Connection to a Server - Use this approach to transfer network data directly from to a network file.  This approach incorporates current network conditions and is suitable for most applications.
  • Importing Data from a File -  This offline data collection approach makes use of a script file to collect data from and store it in a file for import at a later time.  This approach is particularly useful for applications where network access is limited.

Connect to a Server

To use this approach to importing data, you must establish a connection to the Oracle database located on your Server.  To establish a connection, click on the Connect to Server button on the Select Data Source tab.  Next, enter the information you use to logon to the Oracle database and click the Connect button.  When you do so, the Modeler will use the information to connect to the server.  When a connection has been established, the Select a User Group tab will appear.

Note:  Most network operators use the default port for Oracle of 1521.  However, in some applications, an alternate port is assigned.  If you are having difficulty establishing a connection, ask your network operator to specify the correct port number.

Import Network Data From - Select a User Group

When a connection to a server is established, the Select User Group tab displays a list of the user groups present on the server.  User groups are established within to enable you to control who has access to the various portions of your network.  By selecting a user group on this tab, you are effectively selecting the subset of network data that you wish to import.  Network objects will be imported based on the following conditions:

  • For locations, the location must be a member of the selected user group or of a sub-group (child) of the user group.
  • For trunks and circuits, both endpoints must be a member of the selected user group or of a sub-group (child) of the user group.
  • For circuit paths, all of the trunks that the path traverses must be a member or of a sub-group (child) of the user group.

Network objects that do not meet the above criteria or are only partially configured (i.e. a circuit without a path) will not be imported.  If errors exist, the Modeler will display an error log upon import completion.  A list of any objects that failed importation will be included in the log.

Select a Import File

Prior to using this approach to importing data, you must create an import file by executing a Unix script (available from ) to collect network data and store it in an xml file.  To import the file you created, click on the Select a Import File button on the Select Data Source tab.  Next, click on the Select button and use the file browser to locate the desired file.  When your file selection is complete, click the Get Data button to proceed to the Browse Network Data tab.

Note: The Select a User Group tab is not applicable when the data source is an import file.

Browse Network Data

The Browse Network Data tab contains a list of data tables obtained from your network data source and allows you to preview the data selected for import.  Using the list, click on the name of a data table to display it's contents in the grid below.  Within the data grid, you can:

  • Rearrange columns by clicking on a column name and dragging it to the desired location.
  • Sort the data using the column of your choice as the index.  To do so, click on the name of the column you wish to sort by.  An arrowhead will appear. Click on the arrowhead to alternate between an ascending or a descending sort order.
  • Copy all or part of the data to the clipboard for use in other applications (i.e. Excel).  To do so, use Cntl-Click or Shift-Click to select the rows of data you wish to copy then use Cntl-C to place the data in the clipboard.

You can revise your selection of data types if necessary.  To do so, return to the Select a User Group tab, adjust your selections, and click the Get Data button.  The Browse Network Data tab will be updated to reflect your changes.

To save a snapshot of the entire set of data tables, click on the Save DataSet button.  To open a previously saved data set file, click on the Open DataSet button.

To proceed to the Select Import Options tab, click the Next button at the bottom of the dialog.

Select Data Import Options

Network Data Import Options

Import Option Description
Nodes When checked, the Modeler will import all SN16000, SN9000, and SN3000 locations.
APS When checked, the Modeler will import any APS/MSP Protection Groups that are present in the network.  APS/MSP protection groups are listed in the Object tree under the location they are assigned to.  This option is available only when the Nodes option is also selected.
PPG When checked, the Modeler will import any Path Protection Groups that are present in the network.  Path protection groups are listed in the Object tree under the demand they are assigned to.
Ethernet When checked, the Modeler will import any Virtual Concatenation Groups that are configured in the network.  Virtual Concatenation Groups appear in the Object tree with the Ethernet Services they connect listed as child objects underneath them.
Mesh When checked, the Modeler will import all BroadLeaf trunks that have both trunk endpoints configured and all Broadleaf circuits including local circuits.  When importing from a live network, any circuits that are not fully operational at the time of import will be brought in as un-routed demands and any trunks that are not fully operational will not be brought in.
Ring When checked, the Modeler will import all 2-Fiber and 4-Fiber BLSR/MS-SPRing rings.  UPSR/SNCP rings are not available for import.
Aggregate Mesh Trunks Into LightPaths When checked, the Modeler will combine trunks of similar attributes (i.e. same StartLocation and EndLocation, Type, AdminCost, VPN, and ConduitIDs) into a single lightpath with a CapacityMinimum equal to the number of aggregated trunks.
Load UnReachable Demand Paths When checked, the Modeler will import demand paths despite the fact that they are nonresponsive. 

Select a Format Network and Begin Importing Data

The Modeler allows you to assign coordinates to imported locations based on the location coordinates present in a network file that you specify.  The coordinates determine the relative position of locations within each network view.  Use this feature to avoid having to reposition locations every time you import data from your network.  If you choose not to select a network file, the Modeler will assign default coordinates, creating a grid of locations within each network view. 

To select a format network, click on the Select button and use the file browser to locate the desired file.  When your file selection is complete, click the Import button to begin the transfer of data that completes the import process.  Selecting a format network is optional.  To transfer data without selecting a format network, avoid the Select button and click only the Import button.  The Modeler will begin importing data and log any errors that occur.

To create a network file that you can use as a format network, follow the steps below:

  1. Perform an import from without selecting a format network.
  2. When the import is complete, the Modeler will display your network data using a default network view named LightPaths.  Position the locales within the LightPaths view as desired.
  3. Save the file, noting the file name.
  4. The next time you import, use the saved file as the source for your location coordinates.
