Ring Cover

The Ring Cover algorithm analyzes the Fiber Layer of the Network and creates Rings, and their associated LightPaths, based on algorithm parameters such as maximum length and number of nodes.  The Ring Cover algorithm remembers all Rings that existed before it was run and will not create duplicates.  Prior to running the Ring Cover algorithm, make sure that you have created a Network with all the desired Locations and Fibers connecting them. 

You can run the Ring Cover algorithm using either default or advanced options.  Default ring cover options are in effect from the moment you install the Modeler.  Their values are carefully selected to provide optimum results for a majority of applications.  Ring cover options retain their default values unless you elect to edit them using the Advanced tab dialog.     

Note:  Advanced ring cover options are reset to their default values upon closure of the Ring Cover dialog.

Advanced ring cover options are helpful in those rare situations where default parameters do not provide the desired results.  Each ring cover option is described in the Advanced Ring Cover Options section of this help file.

Once you have determined which options to use, follow the steps below to initiate the ring creation process:

  1. Create any additional objects or object properties necessary to define your network including locations connected by fibers.
  2. Select Algorithms, Rings, Ring Cover from the main menu. The following dialog will appear displaying an empty log:

Ring Cover Dialog, Basic Tab

  1. If you want to execute the algorithm using default ring cover options, proceed to the next step.  Otherwise, click on the Advanced tab and select the desired advanced ring cover options.  When complete, click to return to the Basic tab.
  2. Click the Calculate button on the Basic tab to execute the ring cover algorithm. 

Upon initiation, the Ring Cover algorithm will perform the tasks listed below for all ring demands that are enabled for routing.  

  • Records all existing rings in the network.
  • Generates candidate rings.
  • Generates candidate rings over parallel fibers, if any exist.
  • Sorts ring candidates by ranking
  • Builds top ranked rings.
  • If enabled, builds rings over any uncovered nodes.

The appearance of network information in the log portion of the dialog is your signal that the ring cover process is complete.

Advanced Ring Cover Options

The Advanced tab of the Ring Cover dialog is shown below.  By making selections in this dialog, you determine which factors take precedence during the ring creation process.  Your selections will also influence the amount of time required for the algorithm to complete it's analysis.  Use the description of each option (included below) to determine the desired settings.  When your selections are complete, return to the Basic tab and click on the Calculate button to initiate the ring creation process.

Ring Cover Dialog, Advanced Tab

A description of each advanced ring cover option is provided below.  Any changes you make to option values are retained until the Ring Cover dialog is closed. 

Nodes in Ring

Enter the maximum and minimum number of nodes desired in each ring.  Note that the minimum number of nodes must be at least 3.

Cycle Type

Select BLSR Ring or P Cycle.

Cycle Ranking

Select the ranking technique used to sort the rings.

Find Rings Over Parallel Fibers

Check if there are parallel fibers between nodes. 

Build Rings

Check if you want to actually build rings, leave unchecked if you want to see how many ring candidates are found without actually building them. 

Cover All Nodes

Check if you want to try to cover all nodes with at least one ring. Note that additional rings may be built beyond the Maximum Ring Count to achieve this. 

Maximum Ring Length

Set a maximum limit for the total length of all ring lightpaths.

Maximum Ring Count

Set a maximum limit for the number of top ranked rings that are to be built.
