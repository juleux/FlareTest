Manual Failure Analysis

Use the Manual Failure Analysis tool when you want to analyze the impact of a set of failures while maintaining control over the order in which failures and repairs occur.  All manual failures are evaluated and repaired based on two assumptions.  The first assumption is that lightpath capacity is fixed.  The second assumption is that non-failed resources of all failed demands can be used during the repair process.

Prior to running this algorithm, the network must contain at least some demand routing information (acquired by running a routing algorithm or via import from Silvx Manager).   The Failure Analysis algorithm uses the working path (for Unprotected and SharedPath demands) and protect path (for OnePlusOne demands) assigned to each demand to analyze failures.  Any un-routed demands in the network are ignored by the algorithm.

The process for performing a manual failure analysis is as follows:

  • Open the Failure Analysis dialog by selecting Algorithms, Mesh, Failure Analysis from the main menu.
  • Use the Failure Scenario Editor to create the scenarios you want to analyze.
  • Click on the Advanced tab to select advanced options for manual failure analysis or skip this step and use the default options.
  • Click to select the Manual tab.  
  • Use the manual controls described below to select the scenarios you want to analyze and step through the desired sequence of failures and repairs.
  • Use the Results tab to evaluate the impact of failures and repairs as you progress through your analysis sequence.  If a network view is open, it will be updated to reflect the status of the network as each change occurs.

Refer to the sections that follow for detailed information on using the Manual Failure Analysis tool to select the scenarios that will undergo failure and repair.

Select Scenarios to Analyze

When you click on the Manual tab, a list of the failure scenarios available for selection appears.  To select a scenario, click on it's name.  Multiple scenarios can be selected by holding down the Cntl or Shift keys while you click on the desired scenario names.  When your selection is complete, click on the >> button.  The scenario(s) you chose will now be appear in the "To Fail" list of scenarios.  To remove a scenario from the selection list, click on it's name in the "To Fail" column followed by the << button.

Failure Analysis Dialog, Manual Tab - Select Scenarios

Fail Scenarios

To perform a manual failure, you must first create a list of the scenario(s) you wish to fail.  When your list is complete, click on the Calculate button at the bottom of the Manual tab in the Failure Analysis dialog.  When you do so, the algorithm will perform the failures listed in the "To Fail" column (along with any selected repairs in the "To Repair" column) and update both the Results tab and the network view.  The list of failed scenarios will move to the "To Repair" column within the Manual tab.  Note that all of the failure scenarios in the "To Fail" list will be failed as a group. 

Failure Analysis Dialog, Manual Tab - Fail Scenarios

Repair Scenarios

To perform a manual repair, you must first fail the scenario(s) that you wish to repair. To select a scenario for repair, click on it's name in the "To Repair" list.  Multiple scenarios can be selected for repair by holding down the Cntl or Shift keys while you click on the desired scenario names.  When your list is complete, click on the Calculate button.  When you do so, the algorithm will perform the selected repairs (along with any failures listed in the "To Fail" column) and update both the Results tab and the network view.  The list of repaired scenarios will return to the "To Select From" column within the Manual tab.  To return the Manual tab to it's original state upon opening, click on the Reset button.
