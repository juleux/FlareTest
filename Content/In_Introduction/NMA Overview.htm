Introduction to the Network Modeling Application

The Network Modeling Application (Modeler, NMA) is a member of the ™ family of network design tools.  Its purpose is to aid designers with the task of converting a set of traffic forecast requirements into a budgetary network design.  The Modeler streamlines the process of finding the best case design - one in which capacity and protection requirements are satisfied and cost is minimized.

The Modeler consists of the following:

  • A network object model for creating mesh and/or ring designs using conduits, locations, fibers, lightpaths, rings, and demands.
  • The ability to import data from a functioning network via .
  • A variety of tools for analyzing, comparing, and editing network data.  Using these tools, designers can vary network parameters and evaluate the effect on performance and cost.
  • A graphical overview (network view) of network information.  The network view is updated throughout the design process to provide immediate feedback on the impact of design changes.
  • A set of sophisticated algorithms to route demands, optimize shared protection, perform failure analysis, create rings, and dimension rings.
  • A variety of reporting tools that can be used to analyze and/or document a design or to export network data to Excel.

Each of the above is described in greater detail throughout this help file.

The Modeler is intended for use by system engineers, channel partners, customer service engineers, and by clients implementing network designs.  A basic understanding of both optical networking and of the Windows environment is required to use this tool. 

To run the Modeler from the Start menu, click on Programs, , then select Modeler.
