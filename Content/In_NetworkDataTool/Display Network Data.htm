Display Network Data

Using the Network Data Tool, you can display detailed information about a set of objects of your choosing.

To select an object and the type of data you want to display, open the Network Data Tool, select the Reports tab, and follow these steps:

  1. Select an Object Type - Use the pull-down menu to select an object type.  When you do so, a list of objects of the type you selected will appear in the Filtered Objects list.
  2. Select a Filter (optional) - If the list of objects to select from is extensive, you have the option of applying one or more filters to pare down the list and make the task of locating objects easier.  Each object type has its own set of filters to choose from.  Examples include group name, type, and start location.  For instructions on applying a filter, read the Filter Network Data section of this help file. 
  3. Select an Object(s) - Click to select an object from the list or select multiple objects by holding down the Cntl or Shift key while you click.  To select the entire list, click on the Select All button at the bottom of the dialog.  To clear your selections, click on the Clear All button at the bottom of the dialog.

Network Data Tool - Select an Object

  1. Select a Report Type - For each type of object, there is a list of report types to choose from.  Refer to the table below for a description of each report type.  The data points included in any one report type vary depending on the type of object (i.e. conduit, location) the data is associated with.  To select a report type, click on the pull-down labeled Report and scroll down to make your selection. 

Network Data Tool - Description of Report Types

Report Type Description Objects Report Applies To
Data Data reports provide descriptive information about each object.  Examples include start and end locations, scope, type, and group. Location, Fiber, LightPath, Ring, Demand, Virtual Concatenation Group
Traffic Traffic reports provide information about the type and amount of traffic flowing over, flowing through, or dropping at each object. Conduit, Location, Fiber, LightPath, Ring
Capacity Capacity reports provide various capacity related data points such as incremental capacity, maximum capacity, and minimum capacity. Conduit, Location, Fiber, LightPath
Hosted Items Hosted Items reports provide a list of the objects that are hosted by (associated with) each object in the report.  For example, the Fiber Hosted Items report contains a list of the lightpaths, rings, and demands that are routed over each fiber included in the report.   Conduit, Location, Fiber, LightPath, Ring, Path Protection Group, Virtual Concatenation Group
Hosts Host reports provide a list of the objects that host each object in the report.  For example, the LightPath Hosts report contains a list of the fibers, conduits, and locations that combine to create working and protect paths for each lightpath included in the report.   Fiber, LightPath, Ring, Demand, Path Protection Group, Virtual Concatenation Group
Ports Port reports provide detailed connection information for each location included in the report.  Examples include connection name, connection type, time slots, chassis, slot, and port numbers. Location
APS/MSP Protection Groups These reports provide descriptive information about the APS/MSP Protection Groups associated with each location included in the report.  APS/MSP Protection Group information is imported from . Location
Path Protection Groups This report provides descriptive information about the Path Protection Groups associated with each location included in the report.  Path Protection Group information is imported from . Location
Virtual Concatenation Groups This report provides descriptive information about the Virtual Concatenation Groups associated with each location included in the report.  Virtual Concatenation Group information is imported from . Location
Time Slots This report contains a list of the time slot assignments for each lightpath included in the report.  For each time slot assignment, the report contains a variety of information that describes the time slot itself, the lightpath and demand assigned to it, and the hardware associated with it.   LightPath
Fiber Routing This report lists the fibers that each lightpath is routed over.  Included is the name of the fiber and it's start and end locations. LightPath
Routing This report lists the time slots, lightpaths, fibers, conduits, locations, and rings associated with each demand's routing path.  It also provides routing statistics for each demand such as AdminCost, Length, Fiber Hops, LightPath Hops, and Latency. Demand
Connections This report provides detailed connection information for each demand included in the report.  Examples include connection name, connection type, time slots, chassis, slot, and port numbers. Demand
Ethernet Services This report provides descriptive information about the Ethernet Services associated with each Virtual Concatenation Group included in the report.  The report includes physical placement information as well as logical and physical address information. VCG and Ethernet Service information is imported from . Virtual Concatenation Group

When these steps are complete, the report data associated with your object selections will appear in the table located on the Reports tab of the dialog (as shown above).  You can sort the data by clicking on the desired column heading.  When you do so, an up-arrow will appear to indicate that the data in the column is listed in ascending order.  Click on the arrow to toggle the sort order between ascending and descending. 

To export the data you've selected, click on the Save to Excel button on the upper-right side of the Reports tab.
