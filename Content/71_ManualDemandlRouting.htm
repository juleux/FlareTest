Open the Manual Demand Routing Tool

To access the Manual Demand Routing tool:

  1. Click on a demand name in the Object tree.
  2. Locate the WorkingPath property (or the ProtectPath property) and click on it.  A button with … in it will appear on the right edge of the box.
  3. Click on the ellipsis … button (as shown below).
  4. The manual routing dialog will appear.  The dialog will display any existing routing information for the demand you selected. 

Property Editor - Edit Demand Working Path

A sample Manual Routing dialog is shown below.  It contains the two lists described below:

  • CurrentLocation LightPaths - Within this tool, current location refers to the location that is the starting point of the current hop.  The CurrentLocation LightPaths list contains the set of the lightpaths that connect to the current location. Lightpaths are ranked top to bottom by cost.  Each time a lightpath is selected, this list is updated to reflect the change in the current location. 
  • Path LightPaths - This list contains the set of lightpaths that have been selected to form the proposed route.  Lightpaths are listed in the order that they are selected.  The list is considered complete when it's start and end points match those of the demand being routed.

Apply Routing and Diversity Constraints

Routing and diversity constraints provide a means of setting priorities for lightpath selection during the routing process.  Potential lightpath selections that violate a hard constraint are not made available for selection in the CurrentLocation LightPaths list.  When soft constraints are specified, a cost penalty is applied to any potential lightpath selection that is in violation. By comparing the combined penalties for each potential selection, you can identify the lightpath that best meets all of your criteria.  Lightpaths that violate soft constraints appear in the CurrentLocation LightPaths list in red text.

The Modeler includes routing constraints for a lightpath's maximum capacity, minimum capacity, VPN, and through traffic requirements.  To select options for routing constraints, click on the Routing Constraints... button.  For a description of each constraint option, read the Routing Constraints section of this help file.

The Modeler's diversity constraints provide a means of maintaining conduit and/or node diversity during the routing process.   To select options for diversity constraints, click on the Diversity Constraints... button. The dialog shown below will appear:

 

The impact each diversity constraint option has on the routing process is described below.

Constrain Path Diversity by Conduit

Select one of these two options:

  • Hard: If routing a given demand over a lightpath will cause the lightpath to share a conduit with another of this demand's paths, then that lightpath will be not be available for selection in the CurrentLocation LightPaths list.
  • Soft:  If routing a given demand over a lightpath will cause the lightpath to share a conduit with another of this demand's paths, then a cost penalty will be assessed on the lightpath and it will be displayed in red in the CurrentLocation LightPaths list.

Constrain Path Diversity by Node

This option is enabled only when the demand's NodeProtection property is set to True.  Select one of these two options:

  • Hard: If routing a given demand over a lightpath will cause the lightpath to share a node with another of this demand's paths, then that lightpath will be not be available for selection in the CurrentLocation LightPaths list.
  • Soft:  If routing a given demand over a lightpath will cause the lightpath to share a node with another of this demand's paths, then a cost penalty will be assessed on the lightpath and it will be displayed in red in the CurrentLocation LightPaths list.

Clear an Existing or Proposed Route

To clear an existing or proposed route, click on the Clear All/Selected LightPaths button.  The entire Path LightPaths list will be deleted.

Clear one or more LightPaths in a Route

Click to select a lightpath from the Path LightPaths list.  Click on the Clear All/Selected LightPaths button. The lightpath you selected, along with any lightpaths that are downstream from it (below it in the list) will be removed from the Path LightPaths list.

Find the Shortest Route from the Current Location

To find the shortest path from the current location to the end location, click on the Find Shortest Path From CurrentLocation button. You can use this button when there is no existing route information or when a partial route is present in the Path LightPaths list.

Create a New Route

To create a new proposed route, perform the following steps:

  1. Set routing and diversity constraints.
  2. Clear existing path information, if it exists. 
  3. From the CurrentLocation LightPaths list, select the desired lightpath and double-click on it.  Lightpaths are ranked based on cost, with the lowest cost selection at the top of the list. Your selection will be added to the Path LightPaths list and it's endpoint will become the new Current Location.  At the same time, the CurrentLocation LightPaths field will be updated with lightpaths that connect to your previous selection. 
  4. Continue to select lightpaths until a path is established that connects the demand's start and end locations.  At any point during the selection process, you can click on the Find Shortest Path From CurrentLocation button. 
  5. When you are satisfied with the path, click on the OK button.  When you do so, the demand's routing path will be updated and the text "Manually Routed" will be added to it's RoutingNotes property.

Edit Form Opacity

Use the Form Opacity slider to reduce the opacity of the manual routing dialog.  This functionality is helpful when you want to see the lightpath selections you are making highlighted within the network view.  The desired network view must be displayed prior to opening the manual routing tool.

Prevent a Route from being Overwritten

When the Fix LightPath Routing option is checked, the FixedLightPathRouting property of the demand being edited will be set to True when you exit the manual routing tool.  By using this default option you prevent the route you have selected from being overwritten the next time a routing algorithm executes. 

When the Fix LightPath TimeSlots option is checked, both the FixedLightPathTimeSlots property and the FixedLightPathRouting property of the demand being edited will be set to True when you exit the manual routing tool.  By using this default option you prevent the time slots assigned during the manual editing process from being overwritten the next time a routing algorithm executes. 

Both the Fix LightPath Routing and the Fix LightPath TimeSlots options are selected by default.  While you may opt to fix lightpath routing without also fixing time slots, you cannot fix time slots without also fixing lightpath routing.  In other words, the Fix LightPath TimeSlots option must be used in conjunction with the Fix LightPath Routing option.

Reset the Manual Routing Tool

When you reset the Manual Routing tool, you return it to the state it was in when you first opened it.  This removes all of the edits you made to the route since opening the tool and restores any route information that was present.  To reset the Manual Routing tool, click the Reset button on the right side of the dialog.

Manual Routing Example

This example will guide you through the process of manually routing the working path of a previously routed demand named D_75.  Prior to opening the tool, you may wish to open a network view to help you visualize potential routes during the manual routing process.  The next step is to select a demand and open the tool.  Upon opening, the Manual Routing tool displays the existing path information for the demand as shown below.  In this case, we want to create a new route that avoids the location Kansas City, which is also present in this demand's protect path. 

Prior to editing the route itself, select the desired settings for both Routing and Diversity constraints.

The next step in the process of creating a new route is to delete all or part of the existing route.  For this example, we will retain the first lightpath in the existing route, and remove the remaining lightpaths.  To accomplish this, click on the second lightpath in the list: LP_Kansas City_St. Louis as shown below.

Next, click on the Clear All/Selected LightPaths button to remove the selected lightpaths as shown below.

The CurrentLocation LightPaths frame now shows St. Louis as the CurrentLocation and the lightpaths listed below it are those that connect St. Louis to other locations in the network. Any lightpaths that violate the hard constraints we selected do not appear in the list. Any lightpaths that violate the soft constraints we selected ( in this case, none) appear in the list in red text.

To add a LightPath to the Path LightPaths list, double-click on the desired lightpath in the CurrentLightPaths list (top half of dialog). In this example if you double-clicked on the LP_Nashville_St. Louis, the dialog would be updated to look like the one below.

Edit Demand Working Path - Add LightPath to Route

Simply repeat the process of selecting lightpaths and double-clicking on them to create the desired route. To better visualize potential paths, you can adjust form opacity to reveal any network views on display beneath the tool.  At any point in the selection process, you can:
  • Click on the Add Shortest Path from CurrentLocation and the Modeler will find the shortest path to the end location for you.
  • Click on the Reset button if you are unhappy with the path the proposed route is taking and want to restart the selection process from the beginning.
  • Remove one or more lightpaths in the proposed route using the Clear All/Selected LightPaths button.

The proposed route is complete when it connects the demand's start location to it's end location as shown below.  Click the OK button to save the proposed route and exit the tool.
