Derive LightPath Properties From Fiber Length

When fiber information is available, the amount of data that must be entered to create a network file can be minimized by deriving values for LightPath Length and LightPath AdminCost properties from Fiber Length.  To accomplish this, follow the steps listed below:

  1. Set Fiber Length
  2. Derive LightPath Length from Fiber Length
  3. Derive LightPath AdminCost from LightPath Length

Each of these steps is described in detail below.

Set Fiber Length

To set Fiber Length, enter a value for the Length property of each Fiber object in your design.  The most efficient means of accomplishing this is to enter length values into the Fibers worksheet of a Network file and open the file in the Modeler (as input).  To use the Modeler's interface to set Fiber Length, click on a fiber in the Object tree to select it and enter a value using the Property editor.

Derive LightPath Length from Fiber Length

To derive LightPath Length from Fiber Length, click on the LightPaths collection object in the Object tree to select it.  Using the Property editor, enter a value of True for the LengthsFromFiber property.  When you do so, the Modeler will calculate a length value for each lightpath by totaling the lengths of the fibers it travels over.  To verify this action, select a lightpath in the Object tree and note the value of its Length property in the Property editor.  

Note: Properties for collection objects often serve a dual purpose.  They provide status information for the collection and provide a means for you to edit collection members as a group.   The LengthsFromFibers property of the LightPaths collection fits this description.  When you set it to True, the LengthsFromFibers property of each LightPath object is set to True.  When it is False, it means that the LengthsFromFibers property for one or more objects in the collection has been set to False.
