Add a Ring

This section describes the process for creating rings using the Modeler's Object toolbar.  The Modeler also provides an algorithm that uses the available data to generate a set of rings for an entire network.  For more information on this algorithm, read the Ring Cover section of this help document.

Rings can be created from a set of fibers or a set of  lightpaths.  To add a ring, you must generate a list of the fibers or lightpaths that will connect to form the ring.  The process varies slightly depending on whether fibers or lightpaths are being used.

 To add a ring to your design:

  1. Using a network view, select the fibers or lightpaths that will form the ring.  Selections can be made using one of the following methods:

 

  • Click and drag a selection box (rubberband) around the desired fibers and/or lightpaths.  LightPaths must be visible within the view in order to select them (the LightPathRouting property of the current view must be set to Direct).  
  • Hold down the Cntl key and Click to select each fiber/lightpath.

 

Note:  You can skip this step if you prefer to make your selections using the Add Ring dialog as shown below.  The Add Ring dialog provides a complete list of fibers or lightpaths to choose from.  To skip this step, click on any object in the network view, then proceed to step 2.

Add a Ring - Select Fibers and LightPaths using Click and Drag

  1. Click on the Add Ring icon in the Object toolbar or select Edit, Add, Ring from the main menu. The dialog shown below will appear.  Enter a unique name for your ring and click OK.

Add a Ring - Enter a Unique Name

  1. Using the Add Ring dialog, select one of two methods for creating rings by clicking on the appropriate tab: From Fibers or From LightPaths. 

The From Fibers dialog is shown below.  On the left is a list of fibers that are available to add to this ring.  Any fibers you selected prior to opening the dialog are listed under Selected Fibers.  Fibers can be transferred between the two lists using the arrow controls at the bottom of the dialog.  When you are satisfied with your list of Selected Fibers, select the ring locations that require Add/Drop capability.  To select an Add/Drop location, click to place a checkmark next to it in the Select Add/Drop Locations list.

The From LightPaths dialog works in the much same way as the From Fibers dialog.  LightPaths can be transferred between a list of available lightpaths and a list of selected lightpaths using arrow controls.  This dialog differs however, in that Add/Drop locations need not be selected - they are derived from the list of selected lightpaths.  

Add a Ring - Add Ring Dialog - From Fibers

  1. A ring will appear in the network view (see below).

Add a Ring - Network View

  1. A new ring object will appear in the Object tree with a list of the lightpaths it contains.  When you select a ring in the object tree, it's properties are displayed in the Property editor.  Modeler gives each new ring a default name and a set of default property values.  Using the Property editor, enter values for the ring's Architecture and Type properties (see below).

Add a Ring - Object Tree and Property Editor

  1. When a ring is created using the From Fibers tab in the ring dialog, Modeler creates a set of lightpaths for the ring.  Modeler gives each new lightpath a default name and a set of default property values.  LightPath Length is given a value equal to the sum of the length of its fibers.  Using the Property editor, enter values for each lightpath's Length, AdminCost, and Type properties (see below).

When a ring is created using the From LightPaths tab in the ring dialog, the selected lightpaths remain in the LightPaths collection.  LightPath Name and other property values remain unchanged.
