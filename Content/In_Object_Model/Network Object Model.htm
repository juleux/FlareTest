The Network Object Model

Understanding the Modeler's object model is key to creating a network design.  The Modeler uses objects to represent the elements of a network.  Users define network and display requirements by creating objects and setting their properties.  The Modeler uses the information contained in objects to fuel its algorithms.

Modeler objects are grouped together to form collections.  Collection objects ( i.e. NetworkViews, LightPaths, All Demands) provide users with information about its group of objects as a whole.  For example, the TotalLength property of the Fibers object contains the total length of fiber in the network.  Collection objects also provide users with the ability to apply properties globally to every object in the collection.  For example, setting the Visible property of a Locales object to true makes every Locale object visible.
