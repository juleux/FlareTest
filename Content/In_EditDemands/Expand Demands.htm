Expand Demands

The Modeler's Expand Demands tool divides large demands into smaller demands based on parameters that you specify.  By routing sets of smaller demands instead of large ones, you create additional routing possibilities and the potential for a better solution. The expansion parameters you specify include:

  • Quantity Threshold - This parameter determines which demands will be expanded.  Demands with a quantity that is greater than this value will be expanded.  Demands with a quantity that is less than or equal to this value will not be expanded.

  • Expanded Demand Quantity - The quantity you want the newly created demands to have.

  • Split Remainder Into Single Quantity Demands - When checked, any value remaining after the expansion calculation is complete will be used to create additional demands, each having a quantity equal to one.  The number of additional demands created will be equal to the remainder.  When left unchecked, one additional demand will be created with a quantity equal to the value of the remainder.  

Demands are expanded using the following calculation:

Demand Quantity/Expanded Demand Quantity = Number of Demands to Create + Remainder (if any).  

To use this tool, select Edit, Demands, Expand... from the menu.  The dialog shown below will appear.  Enter the desired values and click Expand to create additional demands based on your entries. Each new demand will appear in the object tree with a default name (name of the original demand with an added numerical suffix).
