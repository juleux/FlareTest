Evaluating Routing Results

Each of Modeler's algorithms provides a snapshot of routing results in the form of a basic log.  Additional information is available by viewing Network Statistics, LightPath Routing Information, Demand Routing Information, UnRouted Demands, and Non-Diverse Demands as described below.  

Network Statistics

Network statistics provide an overview of a network and its routing results.  To view network statistics, click on the View Network Statistics icon or select Tools, Statistics from the menu.  The dialog shown below will appear.

LightPath Routing Information

To view routing statistics on lightpaths as a group, click on the All LightPaths object in the Object tree and display it's Statistics properties in the Property editor.

To view routing information for a specific lightpath, click on the desired lightpath in the Object tree and display it's properties in the Property editor.  Some of the lightpath properties used to evaluate routing results are described in the table below.

Group Name Description
Time Slots

(applies to Mesh lightpaths only)

 Available The quantity of time slots that were not assigned during the routing process, expressed as a traffic vector.
ProtectDedicated The quantity of time slots set aside for dedicated protect traffic, expressed as a traffic vector.
ProtectShared The quantity of time slots set aside for shared protect traffic, expressed as a traffic vector.
Working The quantity of time slots set aside for working path traffic, expressed as a traffic vector.
Traffic TrafficAvailable The amount of traffic available for routing on a lightpath in units specified by the lightpath's TrafficUnits property.
TrafficMaximum The maximum amount of traffic that can be routed over a lightpath in units specified by the lightpath's TrafficUnits property.
TrafficProtectDedicated The amount of dedicated protect traffic routed over a lightpath in units specified by the lightpath's TrafficUnits property.
TrafficProtectShared The amount of shared protect traffic routed over a lightpath in units specified by the lightpath's TrafficUnits property.
TrafficTotal The total amount of traffic routed over a lightpath in units specified by the lightpath's TrafficUnits property.
TrafficTotalUsed The sum of the amount of protect shared, protect dedicated, and working traffic in units specified by the lightpath's TrafficUnits property. 
TrafficUnits Traffic units are determined by the value of the network's Type property (SONET or SDH).  Units for SONET networks are STS-1/DS-3.  Units for SDH networks are VC4. 
TrafficWorking The amount of working path traffic routed over a lightpath in units specified by the lightpath's TrafficUnits property.
Utilization The percentage of lightpath capacity in use.

Demand Routing Information

To view statistics on demands as a group, click on the All Demands object in the Object tree and display it's Statistics properties in the Property editor.

To view routing information for a specific demand, click on the desired demand in the Object tree and display it's properties in the Property editor.  Some of the demand properties used to evaluate routing results are described in the table below.

Group Name Description
Path-Protect, Path-Working Path Contains a reference to the path's start and end location, as well as the demand it is assigned to.  (Use a network view for a bird's eye view of the path.)  Click on the ellipsis button to manually edit the path.
  PathAdminCost The combined admin costs of the lightpaths over which a demand is routed.
  PathFiberHops The number of fibers over which a demand is routed.
  PathFixedLightPathRouting True/False.  A value of True indicates that the set of lightpaths over which the demand is routed is fixed and cannot be changed by a routing algorithm. 
  PathFixedLightPathTimeSlots True/False.  A value of True indicates that the set of timeslots allotted to this demand is fixed and cannot be changed by a routing algorithm. 
  PathLatency The sum of the propagation delays (in msec) of the lightpaths over which a demand is routed.
  PathLength The sum of the lengths of the lightpaths over which a demand is routed.
  PathLightPathHops The number of lightpaths over which a demand is routed.
  PathStartingTimeSlots

(applies to mesh demands only)

 Contains the starting number of each time slot assignment for a demand.  Each path over which a demand is routed may contain one or more lightpath hops.  During the routing process, a set of time slots is assigned to each hop in the path.  The number of time slot assignments in each set is equal to the quantity (number of circuits of a specified type) of the demand being routed. Starting time slot information is displayed in the following format:  Starting Time Slots for Hop1 (one for each demand quantity, separated by colons), Starting Time Slots for Hop2, ... An example is shown below.

Note that for the Protect path, starting time slots are assigned only when the demand's ProtectionType property is equal to OnePlusOne.  For unprotected demands, there are no time slot assignments and this field remains blank.  For shared path demands, the correct quantity of time slots are set aside, but are not assigned (this approximates real world conditions) and starting time slots are set to zero.
Path-Protect ProtectToWorkLengthRatio The ratio of protect path length to working path length.
Routing Diversity CommonConduitList Contains a list of every conduit that has both the working and protect path of a demand routed over it.
  CommonLocationList Contains a list of every location that has both the working and protect path of a demand routed through it.
  ConduitDiverse Yes/No.  A value of Yes indicates that the working and protect paths for the demand do not share a conduit.
  Diverse Yes/No.  A value of Yes indicates that the working and protect paths for the demand do not share a conduit or a node.  Non-Diverse demands are listed in the Results window.
  NodeDiverse Yes/No.  A value of Yes indicates that the demand has node protection and it's working and protect paths do not share a node.
  VPNViolated True/False.  A value of True indicates that one of the demand's paths (working or protect) is not routed within the demand's VPN.  For this to occur, the By VPN Routing Constraint must have been set to Off or Soft when routing.

Non-Diverse Demands

A demand is non-diverse if it has protection (it's ProtectionType property is set to OnePlusOne or Shared) and it meets at least one of the following criteria:

  • The demand's working and protect paths share a conduit.
  • The demand has node protection (it's NodeProtection property is True) and it's working and protect paths share a node.

When non-diverse demands are present in your network, the Non-Diverse Demands tab will appear in the Results window.  Click on the NonDiverse Demands tab to view a list of the demands that are not diversely routed.
