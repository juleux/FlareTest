Object Descriptions

Conduit:  A conduit is a number that you can assign to group of fibers and/or lightpaths as a means of stating that they share a common point of failure.  What constitutes a point of failure is strictly up to you.  Examples include electrical conduit, ports, cards, and shared power sources. 

Demand:  A group of point-to-point circuits of user-defined type, quantity, and protection.  During the design process, demands are routed over a sequence of lightpaths.

Ethernet Service: 

The connection of a standard Ethernet interface to a Sycamore optical switch.

Fiber:  An individual fiber or a set of interconnected fibers that work as one.  Fibers have user-defined start and end locations. Networks can be defined and routed with or without fiber information.

Locale:  A graphical representation of a location.  Within each network view, there is one locale object for every location object in the network.  Users determine which locales are visible and their X-Y coordinates within the view. 

Local Demand:  A demand that starts and ends at the same location. Also referred to as a local circuit. 

Location:  A unique physical location where one or more nodes exist.

LightPath:  A set of fibers or conduits over which a signal travels.  LightPaths travel from a start location to an end location over a user-defined sequence of fibers or conduits.

Network View:   A graphical representation of a network.  A network view contains a map of interconnected nodes with fiber, lightpath and/or demand information superimposed on top.  Network views can be configured to display various types of network data (i.e. traffic, capacity, routing).  Network views are covered in more detail in the Network Views section of this help file.

Ring:  A set of interconnected lightpaths that form a closed loop.

Virtual Concatenation Group (VCG):  A logical (rather than physical) combination of one or more circuits within a network that provide paths between two endpoint locations.  To each endpoint location, the circuits assigned to a VCG function as a single Ethernet connection.

Virtual Private Network (VPN):  A set of fibers and/or lightpaths that combine to form a private sub-network within a larger network.
