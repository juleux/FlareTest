Display Network Comparison Results

To get started with the Network Compare tool:

  1. Click on the Compare Networks icon or select Tools, Compare... from the main menu.  The dialog shown below will appear.  The name of the network that is currently open will be listed as Network A.
  2. Select Network B, the network you wish to compare the current network to, by clicking on the ellipsis button and selecting the desired file. 

When these steps are complete, the dialog will display the comparison results.  Comparison results consist of lists of objects unique to each network, a list of objects common to both networks, and a list of the mismatched properties of objects common to both networks.  

Network Compare Dialog

Unique Objects

When an object is present in Network A and not present in Network B, it appears in the Objects Unique to Network A list.  Likewise, when an object is present in Network B and not present in Network A, it appears in the Objects Unique to Network B list.  The Modeler updates these two lists whenever you apply an object filter or a property filter using the controls at the bottom of the dialog.

Common Objects

When an object of the same name exists in both networks, it is referred to as a common object.  Common objects appear in the Common Objects list.  While common objects must share the same name, the values of their remaining properties may differ.  A common object whose property values differ between networks is called a mismatch.  Mismatches are highlighted with the symbol.   A common object whose property values are the same in both networks is called a match.  Matches are highlighted with the symbol.

You can use the Show control at the bottom of the dialog to filter the Common Objects list.  To select a filter, click on the type of results (matches, mismatches, or all) you want to include in the list. 

The Modeler updates the Common Objects list whenever you apply a filter using the object filter, property filter, or show control at the bottom of the dialog.

Mismatches

When a mismatch (as indicated by this symbol: ) is selected from the list of Common Objects, the Mismatches list (top right) displays a list of the properties that differ between the two objects.  The name of each property is listed along with it's value in each network.  Using the property filter control, you can filter out properties that you want to exclude from the comparison.  When all of the properties by which two mismatched objects differ are filtered out, the status of the pair will change from a mismatch to a match.
