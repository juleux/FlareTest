Derive LightPath Length From Great Circle Distance

The distance between the locations in a network (i.e. fiber length or lightpath length) must be known in order for the Modeler's algorithms to function properly.  Use this command to calculate the length of the lightpaths in a network based on the coordinates (latitude and longitude) of it's locations.  Use of this command is valid only when there is a direct physical link between the start and end locations of the lightpath. In other words, using this command when express lightpaths are present will yield inaccurate results.  When express lightpaths are in use, first derive fiber length from great circle distance and then derive lightpath length from fiber length.

When you execute the Derive LightPath Length From Great Circle Distance command, the Modeler calculates the great circle distance between each pair of locations in the network and sets lightpath length equal to it.  Great circle distance is the distance between two locations on the earth's surface.  The Modeler calculates great circle distance using the Latitude and Longitude property values for each location as input.  The curvature of the earth's surface is factored into the calculation.

Using this command, lightpath lengths can be set for all of the lightpaths in the network or for a group of lightpaths.  To use this command:

  1. Within the Object tree, click on the All LightPaths object or on the name of the lightpath group you want to work with. 
  2. Under the Commands heading in the Property Editor, locate the LengthsFromGreatCircleDistance property and select True from the pull-down menu.
