Failure Analysis Results

The Modeler provides detailed failure analysis results using a set of tables that work in conjunction with a network view. Open any network view that displays lightpath information prior to running the Failure Analysis algorithm.  After running the algorithm, click on the Results tab.  Display the network view alongside the results tables by selecting Window, Tile Horizontally from the menu. When you select an object (i.e. demand, lightpath) from one of the tables, it's corresponding lightpaths will be highlighted in the network view.

Failure Analysis Results - Results Table and Network View

When you navigate back and forth between the failure analysis dialog and the rest of the user interface, take care not to edit the objects you are viewing.  In order for the data displayed in the dialog to remain synchronized with the data displayed in the rest of the user interface, you must close the dialog prior to making any edits to objects in the network.

The results tables contain the following information:

  • Relevant statistics for the lightpaths evaluated by the algorithm.  Capacity and utilization information is included.
  • Relevant statistics for the failure scenarios evaluated by the algorithm.  Includes a description of the failure and a summary of the resulting demand failures.
  • Detailed information about the results of each failure scenario including:
  • Affected lightpaths - for each failure scenario, a detailed list of the lightpaths affected by the failures.
  • Iteration details - for each failure scenario, a list of results for each iteration of the analysis.
  • Affected demands - for each failure scenario, a detailed list of the demands affected by the failures.
  • Path information - for each affected demand, a detailed list of the paths used to restore it.
  • Affected Virtual Concatenation Groups - for each failure scenario, a list of the VCGs affected by the failures.

A description of how to access each type of results information follows.  To export all of the results information to Excel, place a checkmark next to the Save All Tables option, and click the Save to Excel button.

Failure Analysis Results - Iteration Details

For each iteration the algorithm performs, demand information is compiled using the categories described below.  

Demand CategoryDescription
Ring Restored Protected DemandsDemands (including unprotected demands) that have been successfully restored by ring protection (one or more rings on a demand's path).
Mesh Restored Protected DemandsDemands that have been successfully restored by dynamic mesh restoration.  Note that dynamic restoration is attempted only on demands whose BroadLeafSourceReroute property is set to On.  For demands that traverse one or more rings, dynamic mesh restoration is attempted only if ring protection fails.  By default, the algorithm restores demands over mesh lightpaths.  The algorithm restores demands over ring lightpaths only when the Permit Restoration Over Rings option is selected on the Advanced tab of the Failure Analysis dialog.
Failed Protected Demands Includes the following:
  • OnePlusOne demands for which no alternate route could be found in response to the failure of both it's working and protect paths.
  • SharedPath demands for which no alternate route could be found, either by ring protection or dynamic mesh restoration.
Failed Unprotected Demands Affected, unprotected demands for which dynamic mesh restoration was not attempted.
Unprotected OnePlusOne Demands OnePlusOne demands which have one available path and one failed path after all alternate routes have been evaluated.

For each type of demand, the resulting volume of traffic is quantified using the following three terms:

TermDescription
# DemandsThe number of demands of a particular type.
Qty DemandsFor a set of demands, the sum of the values of their Quantity properties.
Traffic DemandsTotal traffic for a set of demands.  Traffic is calculated (in units defined for the network as a whole) using the demand's Quantity and Type properties.

Path Information for Affected Demands

To view path information for a specific demand, locate the desired demand (the Restored property must be non-zero for a restoration path to exist) within the list of Affected Demands and click on the plus sign to the left of it.  Select Paths from the pop-up list.  A sample list of Paths is shown below.  When you click to select a path from the list, both the failed and the restored paths will be highlighted within the network view.  When you click on the Save to Excel button, all of the Path data within the results tables will be exported to an Excel file.

Note:  By default, paths are not included as part of the results.  To include path information in the results, click on the Advanced tab in the Failure Analysis dialog and place a checkmark next to Show Paths.
