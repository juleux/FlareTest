Assign Demands to a Virtual Concatenation Group

While a virtual concatenation group (VCG) can have multiple demands assigned to it, a demand can only be assigned to a single virtual concatenation group.

In order for a demand to be assigned to a Virtual Concatenation Group, the following criteria must be met:

  • The demand's Type must be compatible with the virtual concatenation group's Structure.  For example, an OC1_STS1_VC3_DS3_E3_AU3 demand is compatible with a STS1_VC3 virtual concatenation group.
  • The demand's endpoints (StartLocation, EndLocation) must be the same as the virtual concatenation group's endpoints (EndPointLocation1, EndPointLocation2) with one exception.  If the virtual concatenation group starts and ends at the same location (EndPointLocation1 = EndPointLocation2), then that location must be identical to either one of the demand's endpoints. 

The Modeler provides two methods for managing the assignment of demands to virtual concatenation groups.  One method entails using the Property editor to select the VCG the demand will be assigned to.  The other method entails using the Assign Demands dialog and works best when you need to edit multiple assignments.

To edit demand assignments using the Property Editor:

  1. Select the demand you want to edit.
  2. In the Property editor, click on the VirtualConcatenationGroup property.
  3. Use the pulldown to select the VCG you want the current demand to be assigned to.  Only VCGs that meet the assignment criteria (described above) will appear in the list.

Property Editor - Setting VirtualConcatenationGroup Property of a Demand

To edit demand assignments using the Assign Demands dialog:

  1. Select Edit, VirtualConcatenationGroups, Assign Demands from the menu.  The dialog shown below will appear.
  2. Using the pulldown, select the Virtual Concatenation Group (VCG) you want to edit.  The Assigned Demands field (right) will list any demands that are currently assigned to the VCG you selected. The Assignable Demands field (left) will list demands that meet the selection criteria (described above).  By default, demands that are assigned to other VCGs are excluded from this list.  To incorporate previously assigned demands into the Assignable Demands list, click to deselect the UnAssigned Demands Only option at the bottom of the dialog. 
  3. Add or Remove Demand Assignments:

    • To assign a new demand to the VCG being edited, click on the desired demand in the Assignable Demands list then click the >> button to move it to the Assigned Demands list.  Use Ctrl-Click or Shift-Click to select multiple demands for assignment.  Any previous assignment a demand had will be automatically removed when a new assignment is made.

    • To remove a demand from the VCG being edited, select the demand from the Assigned Demands list and click on the << button to move it to the Assignable Demands list.  Use Ctrl-Click or Shift-Click to select multiple demands for removal.

  4. Click on the Assign button to finalize your edits and leave the dialog open for further assignments.  Click on the Assign and Close button to finalize your edits and close the dialog.
