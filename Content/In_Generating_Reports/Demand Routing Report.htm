Demand Routing Report

This report is provided as a worksheet entitled "Demand Routing".  It contains the information described in the table below.  Use this report to evaluate routing for one or more demands in your proposed design.

Type of Data Column Name(s) Description
Demand

    

    

    

    

  

  

  

 Name, LongName, Locked, StartLocation, EndLocation This is user supplied information that uniquely describes each demand.
Local True if the demand starts and ends at the same location.
Scope This is user supplied input for demand scope.  Valid entries include:
  • Mesh 
  • Ring 
VPN The virtual private network (VPN) this demand is assigned to.
Quantity Demand size specified by user.  Units are based on Type entry below.
Type This is user supplied input for demand type.  Valid entries include: 
  • AU3, DS-3, VC3, VC4, E3
  • STM-1, STM-4, STM-16, STM-64
  • STS-1, STS-3, STS-12, STS-48, STS-192
  • OC-1, OC-3, OC-12, OC-48, OC-192
Protection Type This is user supplied input for demand protection.  Valid entries include: 
  • Unprotected (working only) 
  • OnePlusOne (working and protect) 
  • SharedPath (working and protect) 
DiverseGroup This is a user supplied property that identifies the name (if any) of the Diverse Group this demand is a member of.   Each demand within a diverse group is routed diversely from the every other demand in the group.
Node Protection True if node protection was specified by the user.  
Broadleaf

  

  

    

 Broadleaf Source Reroute On if Broadleaf Source Rerouting was specified by the user. 
Broadleaf Trunk Restoration On if Broadleaf Trunk Restoration was specified by the user. 
Broadleaf Trunk Reversion User specified.  Valid entries include: 
  • Disabled 
  • Manual 
  • Auto 
Description A user supplied description for each demand. 
Demand Routing

  

  		 Routing Notes Contains text messages from routing algorithms.
Fixed Routing If true, routing algorithms did not change existing routing.
Fixed Timeslots If true, routing algorithms did not change existing timeslot assignments.
Source, Destination Chassis, Slot, Port Identifies the hardware allocated to a demand's source/destination location.
Starting TimeSlot The Starting TimeSlot allocated to a demand's source/destination.
LightPaths Working, Protect The sequential list of lightpaths (comma delimited) over which a demand is routed.
Working Starting Timeslots, Protect Starting Timeslots The sequential list of the timeslots (comma delimited) over which a demand route starts.
Fibers 
(included if fibers are present in the network)		 Working, Protect The sequential list of fibers (comma delimited) over which a demand is routed.
Conduits

  

  		 Working, Protect The sequential list of conduits (colon delimited) over which a demand is routed.
Common The list of conduits (colon delimited) shared by the working and protect paths.
Conduit Diverse Yes if demand working and protect paths are diverse. The working and protect paths for a conduit diverse demand do not share any conduits.
Intermediate Locations

  

  		 Working, Protect The sequential list of intermediate locations (colon delimited) over which a demand is routed.
Common The list of intermediate locations (colon delimited) shared by the working and protect paths.
Node Diverse Yes if demand working and protect paths are diverse. The working and protect paths for a node diverse demand do not share any nodes.
Rings 
(included if rings are present in the network)

   

 Rings The list of rings (comma delimited) over which a demand is routed.
Ring Count The number of rings over which a demand is routed.
Length

   

 Work, Protect The length of each type of demand route.
Protect/Work The ratio of protect path length to working path length.
Latency Work, Protect An estimate of the end to end delay (in msec) seen by a circuit.  The estimate is derived from the network property values of LengthUnit (kilometers or miles) and RefractiveIndex, combined with the Speed of Light.  The value you enter for a refractive index should be representative of the overall index of the fiber in your network.
Protect/Work The ratio of protect path latency to working path latency.
Fiber Hops

   

 Work, Protect The number of fibers over which a demand is routed.
Protect/Work The ratio of protect path fiber hops to working path fiber hops.
LightPath Hops

   

 Work, Protect The number of lightpaths over which a demand is routed.
Protect/Work The ratio of protect path lightpath hops to working path lightpath hops.
STS1 Equiv The number of lightpath hops in STS1 units (as opposed to Type units above) over which a demand is routed.
Equiv LightPath Hops Work, Protect The number of lightpath hops in STS1 units (as opposed to Type units above) over which a demand is routed.
Protect/Work The ratio of protect path equivalent lightpath hops to working path equivalent lightpath hops.
Admin Cost Work, Protect The combined admin costs of the lightpaths over which a demand is routed.

To generate a report, click on the Export Excel Report icon , or select Tools, Reporting from the menu. Place a checkmark next to the type of report you wish to create, then click OK.
